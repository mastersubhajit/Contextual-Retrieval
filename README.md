# A6: Naive RAG vs Contextual Retrieval
![RAG vs Contextual RAG](https://drive.google.com/uc?export=view&id=1sBLlFZNiRV5H8mjCXfY7zSFeT83y5Yqj)

## Overview

This project builds a domain-specific QA system for Chapter 8 of the NLP textbook and compares two retrieval strategies: Naive RAG and Contextual Retrieval. It uses BAAI/bge-small-en-v1.5 for embeddings and Groq-hosted Llama models for both chunk enrichment and answer generation.

| Component | Detail |
|---|---|
| Source | `transformer_jan26.pdf` (Chapter 8) |
| Embedding Model | `BAAI/bge-small-en-v1.5` (HuggingFace) |
| Answer LLM | `llama-3.3-70b-versatile` (Groq) |
| Enrichment LLM | `llama-3.1-8b-instant` (Groq) |
| Chunking | 500 words, 50-word overlap |
| QA Pairs | 20 |

Open and run `A6_RAG_notebook.ipynb` in Jupyter:

```bash
cd A6/

```

The notebook will:
1. Extract and clean text from `transformer_jan26.pdf`
2. Chunk the text (500 words, 50 overlap)
3. Build both vector databases (Naive and Contextual)
4. Run all 20 QA pairs through both pipelines
5. Compute ROUGE-1, ROUGE-2, ROUGE-L scores
6. Save `answer/response-st-125998-chapter-8.json`

## Methods

### Naive RAG
1. Chunk raw PDF text
2. Embed each chunk with `BAAI/bge-small-en-v1.5`
3. For a query: embed query, compute cosine similarity, retrieve top-5 chunks
4. Pass retrieved chunks and question to the LLM to generate an answer

### Contextual Retrieval (Anthropic, 2024)
1. For each raw chunk, generate a 1-2 sentence context prefix describing where it fits in the document
2. Prepend the context and embed the enriched chunk
3. Use the same retrieval and generation pipeline as Naive RAG

**Key insight:** Context prefixes improve embedding quality by adding high-level semantic context, which can improve retrieval accuracy.
